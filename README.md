# 🎓 School Bot

Телеграм-бот + веб-панель для школы: родители регистрируются в боте, выбирают
классы своих детей и получают новости только по своим классам (с картинками).
Учителя/завучи ведут новости и список родителей через простую веб-админку.

Сделано максимально просто — рассчитано на то, что админка может быть в руках
человека старшего поколения, который не хочет разбираться в технических деталях.

---

## 🚀 Возможности

**Для родителей (бот в Telegram):**
- `/start` → ввести имя → выбрать школу (если их несколько) → отметить галочками
  один или несколько классов сразу (например 1-й **и** 3-й) → готово.
- Постоянное меню снизу: **«Мої класи»** и **«Додати клас»** — команды печатать
  не нужно вообще.
- Новости приходят автоматически, только по тем классам, которые выбрал родитель.
- Если новость отредактировали (например, исправили ошибку или поменяли
  картинку) — сообщение в Telegram обновится само, второй раз присылать не надо.

**Для администратора (веб-панель `/admin`):**
- **Школы / Классы** — справочники.
- **Родители** — появляются в списке сами, как только зарегистрировались в боте.
  Создавать вручную не нужно. Можно найти по Telegram ID или имени и
  поставить галочку «Заблоковано» — такому родителю новости больше не придут.
- **Новости** — заголовок, текст, школа, классы (если классы не выбрать —
  новость уйдёт всем классам школы) и картинка — её просто **прикрепляют
  файлом**, ссылку искать и вставлять не нужно.
- Суперадмин видит всё; обычный школьный админ — только свою школу и своих
  родителей.

## 🛠 Установка

```bash
git clone <ваш-репозиторий>
cd school_bot
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Откройте `.env` и заполните:

```env
BOT_TOKEN=токен_от_@BotFather
SECRET_KEY=любая_длинная_случайная_строка
DATABASE_URL=sqlite+aiosqlite:///./school.db
```

> `SECRET_KEY` нужен для подписи сессий в админке — придумайте любую длинную
> случайную строку и никому не показывайте.

Накатите миграции (создаёт файл `school.db` со всеми таблицами):

```bash
alembic upgrade head
```

## ▶️ Запуск

Нужны **два отдельных процесса** — бот и веб-панель. Запускайте их **от
одного и того же пользователя** (без `sudo`), иначе база данных станет
недоступна для записи одному из процессов (см. раздел «Проблемы» ниже).

```bash
# Терминал 1 — телеграм-бот
python main_bot.py

# Терминал 2 — веб-админка
uvicorn main_web:app --reload
```

Админка откроется на http://127.0.0.1:8000/admin

При первом запуске автоматически создаётся суперадмин:

```
логин:  admin
пароль: admin
```

**Обязательно смените этот пароль** после первого входа (раздел «Користувачі» /
«Users» в админке) — иначе в админку может зайти кто угодно.

| `main_bot.py`     | Точка входа бота (long polling) |

## 🩹 Если что-то не так (FAQ)

### `attempt to write a readonly database`
Бот не может писать в `school.db`. Почти всегда это права на файл — например,
админку один раз запустили через `sudo`, и файл стал принадлежать `root`.

```bash
sudo chown $USER:$USER school.db
chmod 664 school.db
chmod 775 .          # SQLite пишет рядом ещё и журнал — папке тоже нужна запись
```
И больше не запускайте процессы через `sudo`.

### `AttributeError: 'BooleanInputWidget' object has no attribute 'validation_attrs'`
Это баг самой библиотеки `sqladmin` 0.27.2: её чекбокс-виджет не совместим с
версией `wtforms`, которую сама же `sqladmin` требует. Падает форма
редактирования родителя (там есть галочка «Заблоковано»).

Уже исправлено в `admin.py` — используется `SafeBooleanField` вместо
сломанного виджета. Если ошибка всё равно появляется — значит, `admin.py` не
обновился, проверьте `git pull` / переустановку зависимостей:
```bash
pip install -r requirements.txt -U
```

### Новости не приходят родителям
Новость приходит **только** тем родителям, у кого выбран хотя бы один из
классов этой новости (или любой класс школы, если в новости классы не
указаны). Если новость не пришла — проверьте:
1. У родителя в боте выбран нужный класс (кнопка «Мої класи»/«Мои классы»
   покажет, что у него отмечено).
2. Родитель не заблокирован в админке.
3. Новость создана для той же школы, где у родителя есть классы.

### После `git pull` что-то не работает
Скорее всего, не накатились новые зависимости или новая миграция БД:
```bash
pip install -r requirements.txt -U
alembic upgrade head
```

### Добавил/изменил поле в `models.py` — что делать?
Сгенерировать миграцию и применить её:
```bash
alembic revision --autogenerate -m "короткое описание изменения"
alembic upgrade head
```
Файл миграции из `alembic/versions/` обязательно коммитить в git — это и есть
история изменений базы, без него у других участников и на проде схема не
обновится.

## 🔒 Безопасность

- Не публикуйте `.env` и `school.db` — там токен бота и данные родителей.
  Оба уже в `.gitignore`.
- Папка `media/` (загруженные картинки новостей) тоже не коммитится.
- Обязательно смените пароль суперадмина `admin/admin` после первого запуска.

## 🩹 Changelog (виправлення цієї збірки)

- `main_web.py` був написаний під стару мульти-тенантну модель (`User`,
  `UserRole`, `SchoolAdmin`, `UserAdmin`, `ParentAdmin` — цих імен вже немає в
  `models.py`/`admin.py`) і не запускався. Переписано під актуальні
  `WebUser`/`WebUserRole` та реальні view з `admin.py`.
- `admin.py`: `StaleDataError` імпортувався з `sqlalchemy.exc`, хоча в
  SQLAlchemy 2.0 він живе в `sqlalchemy.orm.exc` — імпорт падав.
- `requirements.txt`: відсутні `passlib`/`bcrypt` (потрібні `auth.py`) та
  `itsdangerous` (потрібен sqladmin для сесій); версія `redis==5.2.1`
  конфліктувала з `aiogram[redis]` (вимагає `redis<5.1.0`); `bcrypt>=4.1`
  несумісний з `passlib==1.7.4` — усі версії зафіксовано на робочих парах.
- `.gitignore`: була помилка `.env.env` замість `.env` — реальний токен бота
  міг потрапити в git. Виправлено + додано `*.db`, `media/`, `__pycache__/`.
- Додано Alembic (`alembic/`, `alembic.ini`) — миграції замість ручних
  `ALTER TABLE`, env.py бере `DATABASE_URL` з `.env` проєкту.
- Видалено порожній невикористовуваний `bot_main.py` (залишок рефакторингу;
  точка входу бота — `main_bot.py`).
